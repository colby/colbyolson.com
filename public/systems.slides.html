OSI levels

Open Systems Interconnection model, 1984

A conceptual model that characterizes and standardizes the communication functions of a telecommunication or computing system without regard to their underlying internal structure and technology.
  1. Application Layer
  2. Presentation Layer
  3. Session Layer
  4. Transport Layer
  5. Network Layer
  6. Data Link Layer
  7. Physical Layer

1. Physical Layer

Transmission and reception of raw bit streams over a physical medium

  • DSL, Ethernet, Fiber

2. Data Link Layer

Reliable transmission of data frames between two nodes connected by a physical layer

  • IEEE 802.2, L2TP, MAC, PPP, MPLS

3. Network Layer

Structuring and managing a multi-node network, including addressing, routing and traffic control

  • ICMP, IPsec, IPv4, IPv6

4: Transport Layer

Reliable transmission of data segments between points on a network, including segmentation, acknowledgement and multiplexing

  • TCP, UDP

5: Session Layer

Managing communication sessions, i.e. continuous exchange of information in the form of multiple back-and-forth transmissions between two nodes

  • TLS, FTP, HTTP, HTTPS, SSH, SOCKS

6: Presentation Layer

Translation of data between a networking service and an application; including character encoding, data compression and encryption/decryption

  • CSS, HTML, XML, JSON, S/MIME

7: Application Layer

High-level APIs, including resource sharing, remote file access, directory services and virtual terminals

  • Rails, Magento, Wordpress

Responsibility

A good hosting provider should deal with the first four, sometimes five, levels.

As a Systems Engineer, I should know how work my way up and down the OSI model in to order to diagnose issues into their respective levels.

It is important to know which level an issue is at so we can communicate with the appropriate people.
